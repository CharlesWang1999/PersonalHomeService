## 新增需求

### 需求:用户登录功能
系统应当允许已注册用户通过邮箱和密码登录。

#### 场景:成功登录
- **当** 用户提交正确的邮箱和密码
- **那么** 系统返回有效的 JWT 访问令牌

#### 场景:错误的密码
- **当** 用户提交正确的邮箱但错误的密码
- **那么** 系统返回错误提示"邮箱或密码错误"，禁止提示具体是邮箱还是密码错误

#### 场景:用户不存在
- **当** 用户提交一个不存在的邮箱
- **那么** 系统返回错误提示"邮箱或密码错误"

### 需求:JWT 令牌发放
系统必须在用户登录成功后发放 JWT 访问令牌。

#### 场景:令牌包含用户标识
- **当** 系统生成 JWT 令牌
- **那么** 令牌 payload 必须包含用户唯一标识 (sub claim) 和过期时间 (exp claim)

#### 场景:令牌过期
- **当** 用户使用过期的 JWT 令牌访问受保护资源
- **那么** 系统返回 401 未授权错误

### 需求:令牌刷新机制
系统应当支持在令牌过期前进行令牌刷新。

#### 场景:使用有效刷新令牌获取新访问令牌
- **当** 用户提交有效的刷新令牌
- **那么** 系统返回新的访问令牌

#### 场景:使用已过期刷新令牌
- **当** 用户提交已过期的刷新令牌
- **那么** 系统返回错误提示"刷新令牌已过期"，用户需要重新登录
