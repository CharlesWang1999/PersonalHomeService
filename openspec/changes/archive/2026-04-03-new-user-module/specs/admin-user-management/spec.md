## 新增需求

### 需求:管理员查询用户列表
系统必须允许管理员分页查询所有用户信息。

#### 场景:成功获取用户列表
- **当** 管理员请求用户列表
- **那么** 系统返回分页的用户列表，包含用户基本信息（不含密码）

#### 场景:分页参数
- **当** 管理员指定 page 和 size 参数
- **那么** 系统返回对应页码和页大小的用户列表

### 需求:管理员查看用户详情
系统必须允许管理员查看指定用户的详细信息。

#### 场景:成功获取用户详情
- **当** 管理员通过用户 ID 请求用户详情
- **那么** 系统返回该用户的完整信息（不含密码）

#### 场景:用户不存在
- **当** 管理员请求一个不存在的用户 ID
- **那么** 系统返回 404 错误提示"用户不存在"

### 需求:管理员启用/禁用用户
系统必须允许管理员禁用或重新启用用户账户。

#### 场景:禁用用户账户
- **当** 管理员对指定用户执行禁用操作
- **那么** 用户的 accountDisabled 标志被设置为 true，该用户无法登录

#### 场景:启用用户账户
- **当** 管理员对指定已禁用用户执行启用操作
- **那么** 用户的 accountDisabled 标志被设置为 false，该用户可以正常登录

### 需求:管理员删除用户
系统必须允许管理员删除指定用户账户。

#### 场景:成功删除用户
- **当** 管理员对指定用户执行删除操作
- **那么** 系统将该用户标记为已删除（软删除），用户数据保留但用户无法登录

#### 场景:删除不存在的用户
- **当** 管理员尝试删除一个不存在的用户
- **那么** 系统返回 404 错误提示"用户不存在"
