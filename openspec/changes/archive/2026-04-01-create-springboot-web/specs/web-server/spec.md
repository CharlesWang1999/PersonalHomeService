## 新增需求

### 需求:Web 服务监听

系统启动后必须自动监听 8080 端口的 HTTP 请求，并正常处理传入的请求。

#### 场景:服务启动成功

- **当** Spring Boot 应用启动完成
- **那么** 应用必须监听 8080 端口
- **那么** 端口必须接受 TCP 连接

#### 场景:根路径响应

- **当** 客户端发送 GET 请求到 `http://localhost:8080/`
- **那么** 服务必须返回 HTTP 200 状态码
- **那么** 响应内容必须是文本格式

### 需求:健康检查端点

系统必须提供 `/health` 端点，用于健康检查。

#### 场景:健康检查返回成功

- **当** 客户端发送 GET 请求到 `http://localhost:8080/health`
- **那么** 服务必须返回 HTTP 200 状态码
- **那么** 响应内容必须包含 "UP" 字符串
